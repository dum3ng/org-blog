2 legacy way

The class keyword is just a syntax sugar to implement the OO programing in JS, while that not change anything the JS language, it is still proto based and functional. 

  function Shape(){

  }
  function Rectangle(){

  }
  // subclass
  Rectangle.prototype = Object.create(Shape.prototype)
  Rectangle.prototype.constructor = Rectangle

You can either use Object.setPrototypeOf(Rectangle.prototype, Shape.prototype) to achive the prototype link, but that operation is slow according to the fullfilment. And since you assign the prototype of Rectangle to another object, the default prototype with a constructor property assigned to itself when it is declared will be aborted, the result is that if you get the constructor property of a new Rectangle, it will be Shape. That behaviour does not effect the real… . If you assert aRect instanceof Rectangle to be true, you will get what you want. The side-effect is the output of toString will use the constructor property, which means that if you log the info of a Rectangle, you will get : 

Shape: .....

So the line: 

    Rectangle.prototype.constructor = Rectangle

will fix that. And if your other functions rely on the constructor property, you should also benifit from that.
