package kivy app for osx

First install pyinstaller: 

$ pip install pyinstaller

Then assume in the main.py directory: 

$ pyinstaller -y --clean --windowed -n App main.py

If there is no resource files, above steps would be enough. If want to collect all the resource files, edit the generated spec file App.spec: 

coll = COLLECT(exe,
               Tree('/path/to/app/'),# add this
               a.binaries,
               a.zipfiles,
               a.datas,
               strip=False,
               upx=True,
               name='App')

Then repackage: 

$ pyisntaller -y --clean App.spec  # no windowed here, seems will not work fine if added

If the packaged app is too large, manually remove the build and dist folder, and repackage use the spec file. And extrally you can exclude some module by add --exclude-module when generating the spec file, or manually add in the excludes key in Analysis of the spec file.

===================================

I have tried this process on one kivy example, which exists in kivy-examples/demo/showcase/. The packagede app can start normally, but when click the show code button, the app crashed. The console show the error message: 

no such file or directory

Pyinstaller seems run the app under /Users/<username> path, and get empyt string if use dirname(__file__). Since the kivy use kivy.resources.resource_find to load screen, the screen transmission did work, but the read file functionability broken since it use the relative path to ~.

Edit the source code in showcase/main.py: 

from kivy.resources import resource_find

# ....
    def read_sourcecode(self):
        fn = self.available_screens[self.index]
        fn = resource_find(fn) # get absolute path
        with open(fn) as fd:
            return fd.read()

Repackage using the spec file, the app should work without problems.
