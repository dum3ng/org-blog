1.3 string format

Use %(<name>)s to capture a value from the flollowing dictionary: 

# format method
"Hello, {0} {1}".format('Mr.', 'Trump')

# list
"Hello, %s %s" % ('Mr.', 'Trump')

# dictionary
"a string with your %(name)s and %(age)s" % {"name":"katty", "age":28}

  #  Can be quoted by double or single quote.
  #   class is =str=
  foo = 'foo' # foo = "foo"

  # format
  bar = '%s, %d' % ('bar', 2) # bar = 'bar, 2'
  baz = '{1} is {0} {2} old.'.format(8, 'Kell', 'month') # baz = 'Kell is 8 month old.'

2 oop

2.1 class variable

variables defined at class level are all class variables. 

  class My:
      x = 3
      y = 4
      def __init__(self):
          self.private = "private"

2.2 instance variable

Since every function contains a self as first parameter, the variables of instance should be defined with a prefix self.

note
class variables are shared between instances and class. But they are not static, and if an instance defined a variable with same name, the new variable will overwrite the class one.

2.3 public and private

There is no access decorator in python, all variables defined are public, and can be accessed. For private variables, you can define them with double low hysens: 

  class Your:
      __x = 3
      def __init__(self):
          self.__private = "private"

There is a convension that those variables are private and can not be accessed directly because they are automatically add a prefix <className> , so the __x and __private be _Your__x and _Your__private.

3 special variables

3.1 name

The __name__ variable is created when the script is imported or executed directly. If the script is executed directly, i.e. as the main process, __name__ is assigned to "main" : 

$ python myscript.py

If the script is imported as a module, then __name__ is assigned as the module name: 

## file myscript.py
if __name__ == "__main__":
  pass
else:
  pass
## another file
import myscript

Reason:

The reason to create such a variable is you can write a module which can be both executed directly and imported, you would definitely not want to execute when just import it.

4 import module

  1. Alwasy use absolute import.
  2. Use dot `.` to explicitly use relative import.

When using absolute import for current package, you can not run the script directly, that will cause an import error. Use module switch instead: 

;;error
$ python mod0.py  

;; use module switch
$ python -m pack.mod0

Relative import: 

## in A.B.C module
import .mod0  ## A.B.mod0
from . import mod1  ## A.B.mod1
from .. import mod2 ## A.mod2

4.1 import from same directory

If one file is the root path,

4.1.1 import a file from another file

File structure is: ``` folder/ a.py b.py ``` Then you can use: ```python ## in b.py import a ```

4.1.2 import a file in another directory

File structure is : ``` folder/ a.py b.py ``` You can add a `_init__.py` to folder to make it as a package, then import it: ```python ## after change ''' folder/ init.py a.py b.py ''' ## in b.py from folder import a ```

4.2 import from different directory

If both the imported file and current file are in different folders, then the file structure should be: ``` packs/ init.py pack0/ init.py a.py pack1/ b.py ``` The parent folder should also be a package, then use: ```python

from packs.pack0 import a ```

warning: You can not use the package name at package root: ``` packs/ init.py pack0/ init.py a.py core.py ``` If one want to use functions in a.py, then `from packs.pack0 import a` will cause an error: ` packs is not a package`.

In this situation, just use `from pack0 import a`, because it is in the root path.
